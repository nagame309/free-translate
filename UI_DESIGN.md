# UI 設計說明 - 翻譯浮動視窗

## 自適應寬度設計

### 寬度規格
```css
min-width: 200px;   /* 最小寬度 - 避免太窄 */
max-width: 500px;   /* 最大寬度 - 符合可讀性標準 */
width: max-content; /* 根據內容自動調整 */
```

### 為什麼選擇這個範圍？

#### 最小寬度：200px
- **中文**：約 11-12 字/行
- **英文**：約 20-25 字符/行
- **目的**：避免單詞或短句翻譯時視窗過窄，影響美觀

#### 最大寬度：500px
- **中文**：約 28-30 字/行
- **英文**：約 60-65 字符/行
- **依據**：符合中文可讀性標準（25-35字/行）
- **目的**：避免太寬導致視線移動距離過長

### 實際效果

根據翻譯內容長度自動調整：

| 翻譯內容 | 實際寬度 | 每行字數（中文） |
|---------|---------|----------------|
| 單詞翻譯 "Hello" → "你好" | ~200px | 11-12 字 |
| 短句 "How are you?" → "你好嗎？" | ~250px | 14-16 字 |
| 中等句子 | ~350px | 20-22 字 |
| 長句子或段落 | 500px | 28-30 字 |

## UI/UX 最佳實踐參考

### 行長度（Line Length）標準

#### 1. **The Baymard Institute** 研究
- 桌面版：50-75 字符（英文）
- 手機版：35-40 字符（英文）
- 中文約為英文的 60-70%

#### 2. **Material Design** 指南
- 理想行長：40-60 字符
- 最大行長：120 字符（特殊情況）

#### 3. **WCAG（Web內容無障礙指南）**
- 建議每行不超過 80 字符
- 有助於閱讀障礙者閱讀

### 字體設計

#### 字體大小
```css
font-size: 14px;
```
- **14px** 是 Web UI 的標準字體大小
- 在高解析度螢幕上清晰易讀
- 適合快速掃視

#### 行高
```css
line-height: 1.6;
```
- **1.5-1.6** 是正文的標準行高
- 提供足夠的行間距，避免擁擠感
- 有助於視線在換行時不迷失

#### 字體選擇
```css
font-family: 'MapleMono NF CN', ...等寬字體;
```
**為什麼選擇等寬字體？**
- ✅ 對齊整齊，視覺舒適
- ✅ 中英文混排時保持一致性
- ✅ 程式設計師友好（如果翻譯程式碼註釋）
- ⚠️ 可能比例字體略占空間

### 間距設計

```css
padding: 12px 36px 12px 16px;
```

- **上下 padding: 12px** - 提供呼吸空間
- **左側 padding: 16px** - 標準內容邊距
- **右側 padding: 36px** - 為關閉按鈕預留空間

### 對齊方式

```css
text-align: justify;
text-justify: inter-character;
```

**兩端對齊（Justified）的優缺點：**

✅ **優點：**
- 左右邊緣整齊，視覺上更專業
- 充分利用空間
- 適合等寬字體

⚠️ **缺點：**
- 短行可能產生過大的字間距
- 需要配合 `inter-character` 優化中文

## 競品分析

### Google Translate 擴充功能
- 固定寬度：~350px
- 字體大小：14px
- 無兩端對齊

### DeepL 瀏覽器擴充
- 自適應寬度：300-500px
- 字體大小：15px
- 左對齊

### 有道詞典擴充
- 固定寬度：400px
- 字體大小：14px
- 左對齊

## 總結

目前的自適應設計（200-500px）：
- ✅ 兼顧短句和長句的顯示
- ✅ 符合國際 UI/UX 標準
- ✅ 提供良好的閱讀體驗
- ✅ 靈活適應不同內容長度

這是一個**平衡實用性和美觀性**的設計選擇。
